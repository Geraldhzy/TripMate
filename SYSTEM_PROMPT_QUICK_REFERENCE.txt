5️⃣  TRIPBOOK THREE-LAYER ARCHITECTURE

   Layer 1: DynamicData (realtime caches)
   ├─ knowledge{}       → destination keys (TTL-based)
   ├─ weather{}         → {current, forecast, _meta}
   ├─ exchangeRates{}   → {from/to pair → rate}
   ├─ flightQuotes[]    → [{id, airline, price_cny/usd, status}]
   ├─ hotelQuotes[]     → [{id, name, price, checkin/checkout, status}]
   └─ webSearches[]     → [{query, summary, fetched_at}]

   Layer 2: UserConstraints (user-confirmed parameters)
   ├─ destination       → {value, cities[], confirmed ✅}
   ├─ departCity        → {value, airports[], confirmed ✅}
   ├─ dates             → {start, end, days, flexible, confirmed ✅}
   ├─ people            → {count, details, confirmed ✅}
   ├─ budget            → {value, per_person, currency, confirmed ✅}
   ├─ preferences       → {tags[], notes, confirmed ✅}
   ├─ specialRequests[] → [{type, value, confirmed}]
   └─ _history[]        → audit trail

   Layer 3: Itinerary (structured trip plan)
   ├─ phase             → 0-7 (8 phases total, auto-labeled)
   ├─ route[]           → ["city1", "city2", "city3"]
   ├─ days[]            → [{day, date, city, title, segments[]}]
   ├─ budgetSummary     → {flights, hotels, meals, total_cny, budget_cny}
   ├─ reminders[]       → pre-departure checklist
   └─ practicalInfo[]   → [{category, content, icon}]
