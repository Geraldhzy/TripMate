║                    MARKDOWN RENDERING (renderMarkdown)                        ║
║                    Input: Raw text (no HTML)                                  ║
║                    Output: HTML string                                        ║
╚════════════════════════════════════════════════════════════════════════════════╝

Step-by-step pipeline (Lines 717-768):

Input: "# Title\n**bold** text"
  ↓
1. escapeHtml()
   "# Title\n**bold** text" → (unchanged, no HTML chars)
  ↓
2. Code blocks: ```lang\n...\n```
   → <pre><code class="lang-js">...</code></pre>
  ↓
3. Inline code: `code`
   → <code>code</code>
  ↓
4. Headers: # / ## / ###
   → <h1> / <h2> / <h3>
  ↓
5. Bold/Italic: ** / *
   → <strong> / <em>
  ↓
6. Links: [text](url)
   → <a href="url" target="_blank">text</a>
  ↓
7. Blockquote: > text
   → <blockquote>text</blockquote>
  ↓
8. Horizontal rule: ---
   → <hr>
  ↓
9. Lists: - item or * item or 1. item
   → <ul><li>item</li></ul> / <ol><li>item</li></ol>
  ↓
10. Tables: |cell|cell|
    → <table><tr><td>cell</td><td>cell</td></tr></table>
  ↓
11. Paragraphs: \n\n splits
    → </p><p>
  ↓
12. Cleanup: Remove excess <br>, <p> around block elements
    → Final HTML

Output: "<h1>Title</h1><p><strong>bold</strong> text</p>"

OPTION 2: Dedicated SSE Event
──────────────────────────────

Backend sends:
  event: option_chip
  data: {"label":"Option 1","action":"select_option","value":"opt1"}

In handleSSEEvent() add case:

  case 'option_chip': {
    const chipEl = document.createElement('button');
    chipEl.className = 'chip';
    chipEl.textContent = data.label;
    chipEl.dataset.action = data.action;
    chipEl.dataset.value = data.value;
    chipEl.onclick = () => handleChipClick(data);
    
    // Append to bubble
    bubble.appendChild(chipEl);
    scrollToBottom();
    break;
  }


OPTION 3: Hybrid (RECOMMENDED)
──────────────────────────────

Use markdown syntax for content (works with streaming), but support rich metadata 
via optional SSE event. Gives flexibility:

- Content flows naturally in markdown stream
- Chips rendered immediately in markdown output
- Optional `chip_metadata` event enriches chips with callbacks/data
- Backwards compatible
