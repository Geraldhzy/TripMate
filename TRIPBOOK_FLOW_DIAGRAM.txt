┌─────────────────────────────────────────────────────────────────────────────────────┐
│ update_trip_info TOOL INPUT → TRIPBOOK METHOD FLOW                                  │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                       │
│ AI Input (to tool):                                                                 │
│   { constraints: {...}, phase: 1, itinerary: {...} }                               │
│            ↓                          ↓                    ↓                         │
│ Tool Processing (update-trip-info.js):                                             │
│   - Validates phase in [1,7]                                                       │
│   - Validates constraint fields                                                    │
│   - Returns structured response                                                    │
│            ↓                          ↓                    ↓                         │
│ Server Apply (server.js):                                                          │
│   tripBook.updateConstraints(constraints)                                          │
│   ├─ Iterates fields: destination, departCity, dates, people, budget, preferences │
│   ├─ If field changed, records to _history with timestamp                         │
│   ├─ Sets confirmed_at auto if confirmed: true                                    │
│   └─ Shallow merges with existing (preserves unmentioned subfields)                │
│                                                                                       │
│   tripBook.updatePhase(phase)                                                      │
│   └─ Sets phase + phaseLabel from PHASE_LABELS array                               │
│                                                                                       │
│   tripBook.updateItinerary(itinerary)                                              │
│   ├─ If route provided, replaces entire route                                      │
│   ├─ If days provided:                                                             │
│   │   - Merges by day number (updates existing day, adds new days)                 │
│   │   - Smart segment merge: preserves existing segments if new.segments empty    │
│   │   - Sorts days by day number                                                   │
│   ├─ If budgetSummary provided, shallow merges                                     │
│   └─ If reminders provided, appends to Set (deduplicates)                          │
│                                                                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────────┐
│ KEY METHODS & OUTPUTS                                                               │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                       │
│ WRITING:                                                                            │
│   updateConstraints(delta)              → modifies this.constraints fields         │
│   updateItinerary(delta)                → modifies this.itinerary fields           │
│   setWeather(cityKey, data)             → adds/updates weather                     │
│   setExchangeRate(key, data)            → adds/updates exchange rate               │
│   addFlightQuote(quote)                 → appends flight, returns ID               │
│   addHotelQuote(quote)                  → appends hotel, returns ID                │
│                                                                                       │
│ READING (for System Prompt):                                                       │
│   toSystemPromptSection()                                                          │
│     → Includes:                                                                    │
│       • buildDynamicDataPromptSection()    (cached weather, rates, searches)      │
│       • buildConstraintsPromptSection()    (confirmed ✅ vs pending ❓)             │
│       • buildItineraryPromptSection()      (phase, route, selected bookings)       │
│                                                                                       │
│ READING (for Frontend):                                                            │
│   toPanelData()                                                                    │
│     → Returns flat object:                                                        │
│       { destination, departCity, dates, days, people, budget, preferences,        │
│         phase, phaseLabel, route, flights[], hotels[], weather, weatherList,      │
│         budgetSummary, daysPlan[] }                                                │
│                                                                                       │
│ SERIALIZATION:                                                                      │
│   toJSON()                              → serializable object                      │
│   fromJSON(json)                        → reconstructs TripBook instance           │
│                                                                                       │
└─────────────────────────────────────────────────────────────────────────────────────┘
