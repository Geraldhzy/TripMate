# AI Travel Planner - 产品需求文档 (PRD)

> 版本: 1.0 | 最后更新: 2026-04-11

---

## 1. 产品概述

### 1.1 产品定位

AI Travel Planner 是一款**对话式 AI 旅行规划助手**，通过自然语言对话帮助用户从零到一完成出境游行程规划。产品以 Web 应用形式提供，核心体验为「聊天即规划」——用户只需描述旅行需求，AI 自动调用工具获取实时信息（机票、酒店、天气、汇率、景点），渐进式构建完整行程方案。

### 1.2 目标用户

| 用户群体 | 典型场景 | 核心痛点 |
|---------|---------|---------|
| 出境游新手 | 第一次规划海外自由行，不知从何下手 | 信息分散、流程不清、怕踩坑 |
| 时间有限的上班族 | 想省时间，不想逐个比价、查攻略 | 规划耗时、信息碎片化 |
| 深度旅行爱好者 | 需要特色活动（潜水、徒步）的专业建议 | 专业信息难获取、定制化需求高 |

### 1.3 核心价值主张

- **一站式规划**: 从需求收集到行程导出，全流程在对话中完成
- **实时数据驱动**: 机票报价、天气预报、汇率换算等均为实时查询，非静态推荐
- **渐进式引导**: 7 阶段方法论，避免一次性信息过载，逐步深入
- **多模型支持**: 支持 OpenAI / Anthropic / DeepSeek 三大 AI 提供商，用户自由选择

## 2. 功能需求

### 2.1 核心功能

#### F1: 对话式旅行规划

**描述**: 用户通过自然语言描述旅行需求，AI 以对话形式引导规划。

**详细需求**:
- 支持中文自然语言输入
- AI 遵循 7 阶段渐进式规划方法论（见 2.3）
- 对话支持 SSE 流式输出，实时显示 AI 回复
- 支持快捷起步模板（如"日本东京京都大阪7天"、"西欧巴黎罗马之旅"等）

**验收标准**:
- 用户输入旅行需求后，AI 在 3 秒内开始流式响应
- AI 能正确识别目的地、日期、人数、预算等关键信息
- 对话过程中自动推进规划阶段

#### F2: 实时工具调用

**描述**: AI 在对话过程中按需调用 8 个专业工具获取实时信息。

| 工具名称 | 功能 | 数据源 |
|---------|------|--------|
| `web_search` | 搜索签证政策、景点信息等 | Web 搜索 API |
| `get_weather` | 目的地天气预报 | 天气 API |
| `get_exchange_rate` | 实时汇率查询 | 汇率 API |
| `search_poi` | 景点/餐厅/酒店地点搜索 | POI API |
| `search_flights` | 机票报价查询 | 航班 API |
| `search_hotels` | 酒店价格查询 | 酒店 API |
| `cache_destination_knowledge` | 缓存目的地基础知识 | AI + Web |
| `update_trip_info` | 更新行程参考书 | 内部状态 |

**验收标准**:
- 工具调用过程在前端以动画形式展示（开始/完成状态）
- 工具结果以人类可读标签显示（如"找到 5 条结果"、"1 USD ≈ 7.2 CNY"）
- 天气/汇率缓存机制避免重复调用（天气 3h TTL，汇率 4h TTL）

#### F3: 行程参考书 (TripBook)

**描述**: 作为一次规划会话的 Single Source of Truth，分 4 层管理行程数据。

| 层级 | 名称 | 内容 | 生命周期 |
|------|------|------|---------|
| Layer 1 | 静态知识 | 国家/目的地背景信息 | 跨行程复用 |
| Layer 2 | 动态数据 | 天气、汇率、机票/酒店报价 | 带 TTL 自动过期 |
| Layer 3 | 用户约束 | 目的地、日期、人数、预算、偏好 | 用户确认后锁定 |
| Layer 4 | 结构化行程 | 路线、每日计划、预算汇总 | AI 逐步构建 |

**验收标准**:
- 约束变更自动记录历史（field/from/to/reason）
- TripBook 数据实时注入系统提示，AI 不重复询问已确认信息
- 前端右侧面板实时展示 TripBook 状态

#### F4: 智能快捷回复 (Quick Replies)

**描述**: AI 回复后自动检测可交互选项，生成可点击的快捷按钮。

**快捷回复分两层**:
1. **预定义模式匹配**: 13 种常见问题模式（出发城市、日期、人数、预算等），命中后展示预设选项
2. **编号列表检测**: AI 回复中的编号列表（2-6 项，每项 ≤ 40 字）自动转为可点击选项

**特性**:
- 支持单选/多选模式
- 支持自定义输入（allowInput）
- TripBook 已有值的字段自动跳过，避免重复提问

#### F5: 行程概览面板

**描述**: 页面右侧实时展示行程关键信息的结构化面板。

**展示内容**:
- 目的地、出发城市、日期、人数、预算
- 规划阶段进度（Phase 1-7）
- 路线（城市间连线）
- 已查询的机票/酒店报价
- 天气信息
- 每日行程概要

#### F6: 历史行程管理

**描述**: 对话历史保存在浏览器本地，支持多行程切换。

**需求**:
- 自动保存对话记录到 localStorage
- 左侧抽屉展示历史行程列表
- 支持切换、新建对话
- TripBook 快照随对话保存/恢复

#### F7: 多模型设置

**描述**: 用户可选择 AI 模型提供商和具体模型。

**支持的模型**:

| 提供商 | 模型 |
|--------|------|
| OpenAI | GPT-4o, GPT-4o Mini |
| Anthropic | Claude Sonnet 4, Claude Opus 4 |
| DeepSeek | DeepSeek V3, DeepSeek R1 |

**配置项**:
- API Key（本地存储，不上传服务器）
- 自定义 Base URL（支持代理/第三方兼容 API）

### 2.2 目的地知识库

#### 内置知识库

| 知识库 | 触发关键词 | 内容 |
|--------|-----------|------|
| 马来西亚 | 马来西亚/吉隆坡/仙本那/沙巴/槟城/兰卡威/马六甲 | 签证、货币、交通、旅游季节等 |
| 潜水活动 | 潜水/diving/PADI/考证/诗巴丹 | PADI 课程、装备、潜点信息 |
| 节假日安排 | 自动注入 | 中国法定节假日具体日期 |

#### 动态知识库

当用户提到新目的地时，AI 自动：
1. 搜索签证政策、货币、交通等基础信息
2. 整理为结构化 Markdown
3. 调用 `cache_destination_knowledge` 缓存到本地 JSON
4. 后续对话直接复用，不重复搜索

### 2.3 渐进式规划方法论（7 阶段）

| 阶段 | 名称 | 目标 | AI 行为 |
|------|------|------|--------|
| Phase 1 | 锁定约束 | 收集并确认用户核心需求 | 提问确认目的地、日期、人数、预算、偏好 |
| Phase 2 | 机票查询 | 获取真实机票报价 | 调用 search_flights，对比展示选项 |
| Phase 3 | 构建框架 | 确定路线和每日主题 | 生成城市间路线、每日概要 |
| Phase 4 | 关键预订 | 推荐酒店和重要活动 | 调用 search_hotels，推荐住宿/活动 |
| Phase 5 | 每日详情 | 细化每天行程安排 | 具体景点、餐厅、交通方式 |
| Phase 6 | 预算汇总 | 整理总预算明细 | 汇总各项花费，对比预算 |
| Phase 7 | 导出总结 | 生成可保存的行程单 | 完整行程 + 提醒清单 |

---

### 2.4 信息来源与可信度

| 信息类型 | 来源方式 | 可信度保障 |
|---------|---------|-----------|
| 签证政策 | web_search（优先官方来源） | 标注来源链接 |
| 机票价格 | search_flights API | 实时报价，标注查询时间 |
| 酒店价格 | search_hotels API | 实时报价 |
| 天气 | get_weather API | 带 TTL 缓存 |
| 汇率 | get_exchange_rate API | 带 TTL 缓存，双币种展示 |
| 景点信息 | web_search + search_poi | 交叉验证，标注来源 |

**价格展示规则**: 所有价格同时标注当地货币和人民币（通过实时汇率转换）。

## 3. 非功能需求

### 3.1 性能

- 首字响应时间: SSE 流式输出，首 token < 3 秒
- 工具调用超时: 单次工具调用上限 30 秒，超时给出最佳估计
- Agent 循环上限: 单次请求最多 10 轮工具调用

### 3.2 安全

- API Key 仅存储在浏览器 localStorage，不持久化到服务器
- API Key 通过请求头 `X-API-Key` 传输（需 HTTPS 环境保障安全）
- 服务端不存储任何用户对话数据

### 3.3 可用性

- 响应式布局: 聊天面板 + 行程面板自适应
- 中文界面，所有 AI 回复使用中文
- 工具调用状态可视化，用户清晰感知进度

### 3.4 兼容性

- 现代浏览器（Chrome/Firefox/Safari/Edge 最新版）
- Node.js >= 18

## 4. 用户流程

```
用户打开应用
    ↓
设置 AI 模型 & API Key
    ↓
输入旅行需求 / 点击快捷起步模板
    ↓
Phase 1: AI 提问确认约束 ←→ 用户通过 Quick Reply 或自由输入回答
    ↓
Phase 2: AI 查询机票，展示对比表格 → 用户选择
    ↓
Phase 3: AI 生成路线框架 → 用户确认/调整
    ↓
Phase 4-5: AI 推荐酒店 + 细化每日行程
    ↓
Phase 6: AI 汇总预算 → 用户确认
    ↓
Phase 7: 导出完整行程单
```
