  LAYER 1: STATIC KNOWLEDGE (cross-session reuse)
  ├─ knowledgeRefs[]        ["Japan", "Thailand"]
  └─ activityRefs[]         ["Scuba Diving", "Hiking"]

  LAYER 2: DYNAMIC DATA (TTL-based cache)
  ├─ weather{}              { "tokyo": { current, forecast, _meta } }
  ├─ exchangeRates{}        { "USD_CNY": { rate: 7.25, _meta } }
  ├─ flightQuotes[]         [{ id: "f1", airline, price_usd, status }]
  ├─ hotelQuotes[]          [{ id: "h1", name, price_per_night, status }]
  └─ webSearches[]          [{ query, summary, fetched_at }]

  LAYER 3: USER CONSTRAINTS (confirmed by AI)
  ├─ destination            { value, cities[], confirmed, confirmed_at }
  ├─ departCity             { value, airports[], confirmed, confirmed_at }
  ├─ dates                  { start, end, days, flexible, confirmed }
  ├─ people                 { count, details, confirmed }
  ├─ budget                 { value, per_person, currency, scope, confirmed }
  ├─ preferences            { tags[], notes, confirmed }
  ├─ specialRequests[]      [{ type, value, confirmed }]
  └─ _history[]             [{ field, from, to, changed_at, reason }]

  LAYER 4: ITINERARY (AI-built plan)
  ├─ phase                  0-7 (0=not started, 1=constraints, 3=itinerary, 5=summary)
  ├─ route[]                ["Tokyo", "Kyoto", "Osaka"]
  ├─ days[]                 [{ day, date, city, title, segments[] }]
  ├─ budgetSummary         { flights, hotels, activities, total_cny, remaining }
  └─ reminders[]            ["Complete Visit Japan Web 3 days before..."]

├─────────────────────────────────────────────────────────────────────────────┤

  1. search_flights         → TripBook.addFlightQuote()
     Input:  { origin: "MFM", destination: "NRT", date: "2026-05-01" }
     Output: { flights: [{airline, price_usd, duration, stops}] }
     Cache: Per-route, no TTL (live prices)

  2. search_hotels          → TripBook.addHotelQuote()
     Input:  { city: "Tokyo", checkin, checkout }
     Output: { hotels: [{name, price_per_night, rating}] }
     Cache: Per-city/date, no TTL (live prices)

  3. get_weather            → TripBook.setWeather()
     Input:  { city: "Tokyo", date?: "2026-05-02" }
     Output: { current: {temp_c, humidity, description}, forecast: [] }
     Cache: 3 hours per city

  4. get_exchange_rate      → TripBook.setExchangeRate()
     Input:  { from: "USD", to: "CNY", amount?: 450 }
     Output: { rate: 7.25, converted, display }
     Cache: 4 hours per pair

  5. web_search             → TripBook.addWebSearch()
     Input:  { query: "日本签证", language: "zh-CN" }
     Output: { results: [{title, url, snippet}] }
     Cache: Per-query deduplication (prevent re-search)

  6. search_poi             → Not synced to TripBook (results-only)
     Input:  { query, location, category }
     Output: { results: [{name, lat, lon, address, phone}] }

  7. cache_destination_knowledge  → TripBook.addKnowledgeRef()
     Input:  { destination: "Japan", content: "Markdown..." }
     Output: { success, message }
     Cache: 30 days per destination

  8. update_trip_info (Core Writer) → TripBook.updateConstraints/Phase/Itinerary()
     Input:  { constraints: {...}, phase: 3, itinerary: {...} }
     Output: { success, updates, message }
     Cache: Immediate TripBook update + SSE broadcast

┌─────────────────────────────────────────────────────────────────────────────┐
│ CONSTRAINTS FLOW: AI INPUT → TRIPBOOK STORAGE → SYSTEM PROMPT              │
├─────────────────────────────────────────────────────────────────────────────┤

  AI Detects User Constraint
    ↓
  AI calls: update_trip_info tool
    { constraints: { destination: { value: "Japan", confirmed: true } } }
    ↓
  server.js runTool() processes result
    ↓
  tripBook.updateConstraints() merges delta
    ├─ Sets confirmed_at timestamp
    ├─ Records change in _history[]
    └─ Stores with confirmed flag
    ↓
  buildConstraintsPromptSection() generates text
    "## 用户已确认信息（勿重复询问）
     - 目的地：Japan（Tokyo·Kyoto）✅
     - 人数：3人 ❓  (pending)"
    ↓
  Injected into next system prompt
    ↓
  AI sees "already confirmed" → stops re-asking
     or sees "pending" → knows what to ask next
