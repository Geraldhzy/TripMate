📊 LOGGER API AT A GLANCE
═══════════════════════════════════════════════════════════════════════════

Core Methods:
  logger.debug(msg, data)      - DEBUG level
  logger.info(msg, data)       - INFO level
  logger.warn(msg, data)       - WARN level
  logger.error(msg, data)      - ERROR level

Advanced Methods:
  logger.child(context)        - Create scoped logger
  logger.startTimer(name)      - Start performance timer
  logger.generateId()          - Generate 8-char ID
  logger.formatDuration(ms)    - Format milliseconds
  logger.truncate(str)         - Truncate to 200 chars

❓ COMMON QUESTIONS
═══════════════════════════════════════════════════════════════════════════

Q: Where are the log files?
A: There are no persistent log files. All logs go to console (stdout/stderr).
   See: LOGGING_ANALYSIS_SUMMARY.md, section "File Storage"

Q: How do I change the log level?
A: Use environment variable: LOG_LEVEL=debug npm start
   See: LOGGING_QUICK_REFERENCE.md, section "Quick Start"

Q: How do I use the logger in my code?
A: const logger = require('./utils/logger');
   logger.info('message', { key: 'value' });
   See: LOGGING_QUICK_REFERENCE.md, section "Logger Usage Examples"

Q: How do I track requests?
A: const reqId = logger.generateId();
   const reqLog = logger.child({ reqId });
   See: LOGGING_REPORT.md, section "Logger Usage in Project"

Q: Can I output JSON format?
A: Yes: LOG_JSON=true npm start
   See: LOGGING_QUICK_REFERENCE.md, section "Output Formats"
