OBJECTIVE
---------
Fix data persistence asymmetry where conversation history was saved but 
TripBook (itinerary) state was lost when loading historical conversations.

PROBLEM IDENTIFIED (Previous Session Analysis)
-----------------------------------------------
✗ Conversations:  Saved to localStorage.tp_trips (persistent) ✓
✗ TripBook:       Only in sessionStorage.tp_tripbook (ephemeral) ✗
✗ Result:         Historical trips showed chat-only, no itinerary panel

ROOT CAUSE
----------
sessionStorage is cleared on page reload by browser design.
No code existed to persist TripBook state with conversation records.

SOLUTION IMPLEMENTED (This Session)
------------------------------------
✅ Add tripBookSnapshot field to each trip record in localStorage
✅ Capture current TripBook state when saving conversation
✅ Restore TripBook state when loading historical trip
✅ Re-render itinerary panel with restored data

HOW IT WORKS
-------------

Scenario 1: Creating a New Trip
  User sends message
    ↓
  AI responds and updates sessionStorage.tp_tripbook
    ↓
  saveTripSnapshot() executes:
    • Read tripBookSnapshot from sessionStorage
    • Store with trip record in localStorage
    ↓
  Trip now contains both messages AND tripBookSnapshot ✓

Scenario 2: Loading Historical Trip
  User clicks history panel → selects trip
    ↓
  loadTripById() executes:
    • Load trip from localStorage
    • Extract tripBookSnapshot
    • Restore to sessionStorage.tp_tripbook
    • Call updateFromTripBook() 
    ↓
  Chat history visible ✓
  Itinerary panel populated ✓
