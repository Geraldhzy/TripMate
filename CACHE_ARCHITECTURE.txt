getAllCachedDests()
{
  const now = Date.now();
  const result = [];
  
  for (const [dest, entry] of destCache) {
    if (now - entry.saved_at <= CACHE_TTL) {
      result.push({
        destination: dest,
        content: entry.content,
        saved_at: entry.saved_at
      });
    } else {
      destCache.delete(dest);  // Clean expired
    }
  }
  
  return result;
}

================================================================================
                         TTL & EXPIRATION LOGIC
================================================================================

CACHE_TTL = 30 * 24 * 60 * 60 * 1000  (milliseconds)
          = 2,592,000,000 ms
          = 30 days

Check Points:
1. loadCacheFromDisk() - Server startup
   ├─ For each entry: if (now - saved_at) > TTL
   │  └─ SKIP loading (don't add to destCache)
   │
   └─ Result: Only entries < 30 days old are loaded

2. getAllCachedDests() - On each prompt build
   ├─ For each entry: if (now - saved_at) > TTL
   │  ├─ destCache.delete(entry)  ◄─ Remove from memory
   │  └─ Don't include in result array
   │
   └─ Result: Expired entries cleaned up lazily

================================================================================
                      COMPARISON: HARDCODED vs DYNAMIC
================================================================================

HARDCODED KNOWLEDGE FILES (prompts/knowledge/*)
├─ Location:      Source code (git tracked)
├─ Files:         malaysia.js, diving.js, holidays.js, methodology.js
├─ Format:        JS modules exporting strings
├─ Size:          ~16 KB total
├─ TTL:           ∞ (permanent)
├─ Update:        Requires code deployment
├─ Trigger:       Keyword detection in conversation
├─ Token Cost:    Always included when triggered
├─ Examples:      Malaysia, Diving, Holidays, Methodology
├─ Reliability:   High (version controlled)
│
└─ Use When:
   ✓ Destination frequently requested
   ✓ Knowledge rarely changes
   ✓ Want human review before deployment
   ✓ Strategic knowledge (your specialty)

DYNAMIC CACHE (data/dest-cache.json)
├─ Location:      Disk file (gitignored, persistent)
├─ Format:        JSON (human readable)
├─ Size:          ~20 KB (7 destinations currently)
├─ TTL:           30 days
├─ Update:        Immediate via cache_destination_knowledge tool
├─ Trigger:       AI tool execution + smart injection
├─ Token Cost:    Only if destination mentioned in conversation
├─ Examples:      France, Italy, Japan, Turkey, Thailand
├─ Reliability:   Medium (graceful degradation on disk failure)
│
└─ Use When:
   ✓ New/less frequent destinations
   ✓ Knowledge from web searches
   ✓ Want updates without deployment
   ✓ Handle any user-mentioned destination
