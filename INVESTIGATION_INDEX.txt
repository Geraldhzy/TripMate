💡 COMMON QUESTIONS & ANSWERS
──────────────────────────────────────────────────────────────────────────────────

Q: Where does the streaming text go?
A: Into the .bubble div, updated via bubble.innerHTML = renderMarkdown(text)

Q: How often is renderMarkdown called?
A: On every token (every few hundred milliseconds during streaming)

Q: Can I just append chips to the bubble?
A: Not recommended - they'd be lost on next renderMarkdown call. Use regex in
   the function instead.

Q: Why not use innerHTML.appendChild?
A: Because renderMarkdown returns a string, not a DOM element. Parsing regex
   is the right approach.

Q: How do I attach click handlers to chips?
A: Query the bubble for .chip elements after rendering and add onclick handlers

Q: Will chips work during streaming?
A: Yes, if using Markdown syntax approach. SSE-only approach would break flow.

Q: Can users see chips being rendered?
A: With Markdown approach, yes - they appear as tokens stream in.

Q: What if user clicks a chip while streaming?
A: You could disable chips during streaming or queue the click.

Q: Should chips be buttons or divs?
A: Use <button> elements - semantic, accessible, better for forms

Q: How do I know when rendering is done?
A: Handle the 'done' event or attach a post-render callback
