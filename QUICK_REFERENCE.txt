═══════════════════════════════════════════════════════════════════════════════
 TRIPBOOK DATA MODEL
═══════════════════════════════════════════════════════════════════════════════

STRUCTURE (4 LAYERS)
─────────────────────────────────────────────────────────────────────────────
Layer 1: Static Knowledge References
  └─ knowledgeRefs[] - Destinations known (e.g., ["日本", "泰国"])
     activityRefs[] - Activities known (e.g., ["潜水"])

Layer 2: Dynamic Data (with TTL metadata)
  ├─ weather{} - By city: { city, current, forecast, _meta }
  ├─ exchangeRates{} - By pair: { from, to, rate, last_updated, _meta }
  ├─ flightQuotes[] - Array with id, route, date, airline, price_*, etc.
  └─ hotelQuotes[] - Array with id, name, city, checkin, checkout, price_*, etc.

Layer 3: User Constraints
  ├─ destination - { value, cities[], confirmed, confirmed_at }
  ├─ departCity - { value, airports[], confirmed, confirmed_at }
  ├─ dates - { start, end, days, flexible, confirmed, confirmed_at }
  ├─ people - { count, details, confirmed, confirmed_at }
  ├─ budget - { value, per_person, currency, confirmed, confirmed_at }
  ├─ preferences - { tags[], notes, confirmed, confirmed_at }
  ├─ specialRequests[] - { type, value, confirmed }
  └─ _history[] - Change log with timestamps

Layer 4: Structured Itinerary
  ├─ phase - Current phase (0-7)
  ├─ phaseLabel - Phase name (e.g., "构建框架")
  ├─ route[] - Cities visited
  ├─ days[] - Daily plans: { day, date, city, title, segments[] }
  ├─ budgetSummary - { flights, hotels, ..., total_cny, budget_cny }
  └─ reminders[] - Action items

METHODS (KEY)
─────────────────────────────────────────────────────────────────────────────
Write:
  - updateConstraints(delta) - Incremental update with history
  - updatePhase(phase) - Set current phase
  - updateItinerary(delta) - Merge days, route, budget, reminders
  - setWeather(cityKey, data)
  - setExchangeRate(key, data)
  - addFlightQuote(quote) - Auto-generates ID
  - addHotelQuote(quote) - Auto-generates ID

Read/Export:
  - toSystemPromptSection() - For system prompt injection
  - toPanelData() - Flattened data for frontend UI
  - toJSON() / fromJSON() - Serialization

 COMMON MISTAKES TO AVOID

1. Forgetting to call initCache() in server.js startup
   → Cache never loads from disk on restart

2. Calling getAllCachedDests() but not injecting it into system prompt
   → Same as current: cached destinations still invisible to AI

3. Using callback-style fs (fs.readFile callback) instead of .promises
   → Syntax inconsistency with project style

   → Fails if directory already exists

5. Not using try-catch around file I/O
   → Server crashes on permission errors

6. Writing cache synchronously instead of async
   → Blocks AI streaming response

7. Forgetting to add /data/ to .gitignore
   → Cache files committed to git (data pollution)

8. Not checking TTL expiry in getAllCachedDests()
   → Returns expired entries to system prompt
