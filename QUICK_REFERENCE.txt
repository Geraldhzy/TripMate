SSE EVENTS CHEAT SHEET
──────────────────────

Event              When                    Data
─────────────────────────────────────────────────────────────
token              LLM streaming           {text}
tool_start         Tool about to run       {id, name, args}
tool_result        Tool finished           {id, name, resultLabel}
rate_cached        Rate tool ok            {from, to, rate, ...}
weather_cached     Weather tool ok         {city, forecast, ...}
tripbook_update    Trip info tool ok       {constraints, itinerary, ...}
quick_replies      (optional) end          {questions: [...]}
done               Always at end           {}
error              Exception thrown        {message}


CLIENT EVENT STREAM TIMELINE
────────────────────────────

FAST ANSWER (no tools):
  token → token → token → done

WITH ONE TOOL CALL:
  token → token → tool_start → tool_result → token → done

MULTI-ROUND (2 rounds):
  token → token → tool_start → tool_result →
  ← NEW ROUND (LLM called again) →
  token → token → tool_start → weather_cached → token → done


KEY INSIGHTS
────────────

1. Messages array is state machine
   - Initial: [system, user message]
   - Grows each round with: assistant message + tool results
   - LLM always sees full history
   - Never cleared

2. Round boundaries are implicit
   - NO "round_start" event
   - Client infers: token end → tool events → token resumes
   - Design: simpler protocol, less overhead

3. Not all tools get special events
   ✅ get_exchange_rate    → rate_cached
   ✅ get_weather          → weather_cached
   ✅ update_trip_info     → tripbook_update
   ❌ search_flights       → just tool_start/result
   ❌ search_poi           → just tool_start/result
   ❌ web_search           → just tool_start/result

4. Limits and timeouts
   - MAX_TOOL_ROUNDS = 10
   - LLM_TIMEOUT_MS = 120000 (120s)
   - TOOL_TIMEOUT_MS = 30000 (30s)
   - delegationCount <= 2

PROVIDER DIFFERENCES
───────────────────

OpenAI:
  - system prompt in messages array
  - tool_calls in assistant message
  - tool results: one message per tool

Anthropic:
  - system prompt separate parameter
  - tool_use in content array
  - tool results: single message with array

But same loop logic for both!


DEBUGGING CHECKLIST
───────────────────

□ Not seeing round 2?
  → Check for token events ending, then tool_start
  → No explicit round event exists

□ Tool result not cached?
  → Only rate/weather/trip have special events
  → Other tools: check tool_result event instead

□ LLM stuck?
  → Check 120s timeout (LLM_TIMEOUT_MS)
  → Check tool 30s timeout (TOOL_TIMEOUT_MS)
  → Max 10 rounds per request

□ Delegation not working?
  → Max 2 delegations per request (line 594)
  → delegationCount tracked (line 565)

□ TripBook not syncing?
  → Save _snapshot from tripbook_update event
  → Pass tripBookSnapshot in next request
