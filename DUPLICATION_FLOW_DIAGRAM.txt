SCENARIO 1: Incomplete AI Response → Duplication Loop
─────────────────────────────────────────────────────

Update 1 (AI sends only new activity):
  Day 1 existing: [A, B]
  Update: {segments: [C]}
  Merge: [A, B] + [C] = [A, B, C] ✓

Update 2 (AI sends same partial data again):
  Day 1 existing: [A, B, C]
  Update: {segments: [B, C]}  ← Why B again?
  Merge: Map contains [A, B, C], adds [B, C]
  Result: [A, B, C] still 3 items

BUT IF Update 2 sends [A, B, C] again:
  Day 1 existing: [A, B, C]
  Update: {segments: [A, B, C]}
  Merge: [A, B, C] ✓ Still correct

BUT IF User doesn't refresh and streaming fails:
  Day 1 display: [A, B, C]
  Update 2 arrives incomplete: {segments: []} ← Empty!
  Merge: "No segments in update" → Keep existing
  Result: [A, B, C] ✓ Still OK

BUT THEN:
  Update 3 arrives with full data: {segments: [A, B, C, D]}
  Merge: Map merges correctly
  Result: [A, B, C, D]

BUT IF the time+title comparison FAILS:
  Update says: {time:"12:00", title:"Lunch"}
  Existing has: {time:"12:00", title:"Lunch", location:"Tokyo"}
  AND UPDATE ADDS a completely new segment with SAME time:
  
  {time:"12:00", title:"Lunch 饺子", type:"meal"}  ← Different title!
  
  Then keys differ:
  "12:00|Lunch" vs "12:00|Lunch 饺子"
  Result: BOTH get added! [A, B, Lunch, Lunch 饺子] ← DUPLICATION!


SCENARIO 2: Streaming JSON Parse Failure
──────────────────────────────────────────

Streaming chunks arrive as:
  Chunk 1: `{ "segments": [`
  Chunk 2: `{ "time": "21:00", "title": "Bar"`
  Chunk 3: `}] }`
  (Truncated or malformed)

JSON.parse() fails on line:
  `try { args = JSON.parse(tc.function.arguments); }`
  
Catches error:
  `catch { args = {}; }`
  
Result: args = {}, update_trip_info gets empty itinerary!
  tripBook.updateItinerary({}) ← NO-OP
  
Display doesn't update, but backend thinks it did.

Next request has stale TripBook state injected in system prompt,
AI tries to add activity again, causes duplication loop.


SCENARIO 3: Multiple SSE Events Race Condition
───────────────────────────────────────────────

Event 1 arrives: daysPlan = [A, B]
Event 2 arrives: daysPlan = [A, B, C]
Frontend processes Event 1: itineraryState.daysPlan = [A, B]
Frontend renders: shows [A, B]
Frontend processes Event 2: itineraryState.daysPlan = [A, B, C]
Frontend renders: shows [A, B, C]

BUT THEN:
Event 3 arrives (older): daysPlan = [A, B]  ← STALE!
Frontend processes: itineraryState.daysPlan = [A, B]
Frontend renders: shows [A, B]  ← C DISAPPEARED!

User sees C disappear, then AI sends update again, C reappears
as duplicate: [A, B, C, C] ← DUPLICATION!


SCENARIO 4: Deduplication Key Mismatch
───────────────────────────────────────

Existing segment:
  {time: "12:00", title: "午餐 (Lunch)", type: "meal"}
  
New segment from AI:
  {time: "12:00", title: "Lunch", type: "meal"}  ← Different title!

Dedup keys:
  Existing: "12:00|午餐 (Lunch)"
  New:      "12:00|Lunch"
  
Keys don't match → Both get added!
  Result: [Arrival, 午餐, Lunch, Bar] ← DUPLICATION!


═══════════════════════════════════════════════════════════════════════════════
  🎯 MOST LIKELY ROOT CAUSE: Incomplete AI Response
═══════════════════════════════════════════════════════════════════════════════

The AI model (especially when optimizing for tokens) may be sending:
  
  ✓ Only the NEW segment in update_trip_info
  ✗ NOT including the existing segments from system prompt

This works ONCE, but if:
1. AI is called again to modify same day
2. Streaming assembly has issues
3. Session state isn't properly restored

The display shows duplication because:
- Backend merge is correct
- Frontend display is correct
- BUT AI keeps getting asked about same segments
- AND merge logic can't deduplicate what it never sees

Solution: Ensure AI response includes ALL segments for the day,
not just the new ones.
