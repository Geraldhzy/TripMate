TASK #11: Fix Quick Reply skip logic for sub-field patterns
════════════════════════════════════════════════════════════════════════════════

Status: ✅ COMPLETED

Problem:
  Multiple Quick Reply patterns share a constraintField but ask about different 
  sub-fields. The skip logic was too coarse — skips all patterns once ANY 
  sub-field is set.

Affected Patterns:
  • 红眼航班 (preferences.notes) — No field specified
  • 同行人员 (people.details) — Skipped by people.count
  • 预算含机票 (budget.scope) — Skipped by budget.value
  • 单人vs总预算 (budget.per_person) — Skipped by budget.value
  • 日期弹性 (dates.flexible) — Skipped by dates.days

Solution:
  File: server.js (lines 424-438)
  
  Changed skip logic to check:
    1. If pattern has subField → check only that specific sub-field
    2. If pattern has no subField → check main value based on field type
  
  Result: Each pattern now skips independently without blocking others

Code Changes:
  Before: All patterns for same field checked together
  After:  Granular checking of individual sub-fields
  
  Key improvement: 
    if (pattern.subField) {
      const val = c[pattern.subField];
      if (val != null && val !== '' && val !== false) continue;
    } else {
      // Type-specific checks for fields without subField
    }

TASK #15: Fix updateConstraints full-replace bug
════════════════════════════════════════════════════════════════════════════════

Status: ✅ COMPLETED

Problem:
  updateConstraints() does full object replacement instead of shallow merge,
  wiping sub-fields on partial updates.

Root Cause:
  Before: this.constraints[field] = newVal;
  Issue:  Completely replaces the constraint object, losing existing sub-fields

Example Failure Scenario:
  1. constraint.budget = { value: '15000', scope: '含机票', currency: 'CNY' }
  2. Call updateConstraints({ budget: { value: '20000' } })
  3. Bug: Result is { value: '20000' } — loses scope and currency!

Solution:
  File: models/trip-book.js (line 164, updateConstraints method)
  
  Changed to shallow merge:
    this.constraints[field] = oldVal ? { ...oldVal, ...newVal } : newVal;
  
  Now preserves existing sub-fields while overlaying new values

Result:
  1. constraint.budget = { value: '15000', scope: '含机票', currency: 'CNY' }
  2. Call updateConstraints({ budget: { value: '20000' } })
  3. Fixed: Result is { value: '20000', scope: '含机票', currency: 'CNY' } ✓

Impact:
  • Partial updates now preserve sub-fields
  • Enables independent updates to different sub-fields
  • Fixes data loss when updating only specific constraint properties

Verification: ✅ Shallow merge logic confirmed in commit c66aa90
